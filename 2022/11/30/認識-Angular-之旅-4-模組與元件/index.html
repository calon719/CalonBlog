認識 Angular 之旅 (4) - 模組與元件

發表於2022-11-30|更新於2022-12-20|Angular

模組（Module）

模組是將相關功能的程式碼封裝在一起，你可將模組想像成是機器人的各個大部位，比如機器人的手，而手裡面又包含了許多的小部位譬如手指頭、手掌等以及相關的動作，例如抓、握等。
而 Angular 的基本構成要素是 NgModule，Angular 應用程式裡面都會有一個啟動應用程式時作為引導的根模組（root module），通常預設名稱為 AppModule，此外也會根據需求包含不同功能的模組。

@NgModule 裝飾器

在 Angular 應用程式中會將模組所用到的元件資訊都定義在在 @NgModule 裝飾器裡面，我們來開啟在剛建立的專案下 src/app/ 目錄底下的 app.module.ts 來看看裡面有什麼東西：

我們在 Angular 應用程式裡建立模組後會透過 @NgModule 去做相關定義：

declarations

我們需要將要在模組裡面使用到的元件都宣告在 declarations 裡面，否則無法使用，在根模組中會有一個根元件通常為 AppComponent，可以看到 Angular 已經幫我們將它加入到 declarations 裡面了。

imports

當我們在模組中需要使用到其他模組時我們會在 imports 裡面去做設定，在這裡我們除了在建立專案時加入的路由相關的模組 AppRoutingModule 外，還幫我們加入了 BrowserModule，BrowserModule，在稍後我們會介紹。

exports

根據最小知識原則，Angular 會將元件限制在同模組裡面，如果在這個模組裡面有元件需要在其他的地方做重複使用，我們會將要對外公開讓其他模組去做使用的元件在 exports 去做設定。

providers

定義我們使用了哪些服務（Service）。

bootstrap

這只會出現在根模組裡面，用來定義我們 Angular 應用程式的根元件，根元件會在啟動應用程式時將 index.html 裡面的 <app-root></app-root> 標籤內容替換成根元件。


BrowserModule 以及 CommonModule

我們在之後會介紹到 ngIf、ngFor 等通用指令，而這些指令都是放在 CommonModule，在使用時會需要匯入。BrowserModule 則是包含了整個應用程式啟動時會需要基礎設施，裡面也包含了 CommonModule，因為它是函蓋了整個應用程式，所以在除了根模組以外的模組會入會出現要求更改成 CommonModule 的錯誤。

元件（Component）

元件是構成 Angular 應用程式的最小單位，在實務上我們希望可以程式碼可以好維護以及重複利用，會將應用程式拆分成許多元件，我們可以把元件想像成是上面機器人手上的手指頭，應用程式就是由這些元件組合而成的，在應用程式中一定會有一個根元件，而一個 Angular 元件包含了：

  • 負責檢視頁面結果的 HTML 範本（HTML template）
  • 負責頁面樣式的 CSS 檔
  • 負責邏輯以及元件操作的 TypeScript 元件檔

@Component 裝飾器

如同模組有 @NgModule 裝飾器定義模組的資訊，元件也有 @Component 裝飾器來定義元件的資訊：

selector

此屬性所定義的 HTML 標籤名稱會在我們在模組或是元件中需要使用時用到，該名稱格式為「前字元 + 元件名稱」，而 Angular 預設前字元是 app，所以如果我們的元件叫做 demo，當需要在範本裡面使用這個元件時，標籤名稱就是 <app-demo></app-demo>。

templateUrl

定義了元件 HTML 範本的檔案{徑，如果直接將範本寫在元件檔裡的話則是使用 template 屬性。

styleUrls

定義元件的 CSS 樣式檔案路徑，直接將樣式寫在元件檔內的話則使用屬性 styles。
