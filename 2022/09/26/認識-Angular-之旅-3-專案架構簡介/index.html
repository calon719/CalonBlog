認識 Angular 之旅 (3) - 專案架構簡介

發表於2022-09-26|更新於2022-12-08|Angular

這章節我們來了解使用 Angular CLI 所建立的專案目錄裡面有什麼：


根目錄檔案及其他目錄

  • .vscode/：Visual Studio Code 相關設定檔。
  • node_modules/：npm 安裝的相關套件都或放在這裡。
  • **src/**：開發中的程式碼、資源放置的地方。
  • .browserslist：針對不同版本的瀏覽器的相關設定檔案，例如加上 CSS 前綴詞等。
  • .editorconfig：告訴編輯器專案風格的相關設定檔。
  • .gitignore：使用 Git 版控時要忽略的檔案設定。
  • angular.json：Angular CLI 的相關設定，例如：專案的進入點、靜態檔案
  • karma.conf.js：Angular 內建單元測試工具設定檔。
  • package-lock.json：紀錄專案實際安裝的 npm 套件來源以及版本號。
  • package.json：專案 npm 的設定檔，裡面 scripts 已經有設定一些自訂指令，可以根據自己的需求再新增自訂指令。
  • README.md：專案說明文件，使用 Markdown 語法撰寫。
  • tsconfig.app.json：TypeScript 延伸相關設定檔，繼承 tsconfig.json 的相關設定，app 指針對 app/ 裡面的所有 TypeScript 進行設定。
  • tsconfig.json： TypeScript 相關設定檔。
  • tsconfig.spec.json：針對單元測試的 TypeScript 相關設定檔。

src 目錄

  • **app/**：開發中大部份的程式碼都會放在這裡，裡面包含整個網頁應用程式的 Component、Modlue、Service。
  • **assets/**：靜態資源放置的地方，例如圖片。
  • enviroments/：環境變數相關設定檔存放的地方。
  • favicon.ico：瀏覽器分頁頁籤上的 icon。
  • index.html：網頁應用程式的首頁與根首頁，也是 SPA 的唯一頁面。
  • main.ts：Angular 的 JavaScript 主要進入點
  • polyfills.ts：當需要支援舊版本或不同的瀏覽器時會使用到的設定檔，使用 Polyfill 來填補缺少的 HTML5、JavaScript APIs。
  • styles.css：整個網頁應用程式共用的 CSS 檔。
  • test.ts：與 main.ts 類似，用於單元測試檔。

app 目錄

  • app.component.css：根元件 CSS 範本檔。
  • app.component.html：根元件 HTML 範本檔。
  • app.component.spec.ts： 根元件單元測試定義檔案。
  • app.component.ts： 根元件主程式。
  • app.modules.ts：應用程式的 NgModule 定義檔，例如匯入模組、元件等。

assets 目錄

  • .gitkeep：因為 Git 進行版控的時候不會將空目錄進行版控，為了讓 assets 目錄可以在加入版控所存在的空內容檔案。

environments 目錄

  • environment.prod.ts：產品正式上線時所使用的環境變數設定檔。
  • environment.ts：環境變數設定檔。

粗字體的是比較常會去做更動的檔案，剩下的可以等需要用到時再慢慢研究。
